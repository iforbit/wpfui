# Project Dev Log (WPF.UI.DirectX)

## 📅 2025-06-23: GraphSeries 아키텍처 전환 및 렌더링 파이프라인 정비

### ✅ 구조 리팩토링 및 주요 변경점

* `GraphItemBase` 완전 폐기 및 `GraphSeries<T>` 기반 구조로 전환
* `GraphControl`과 `D3D11Renderer`는 더 이상 GraphItem을 직접 다루지 않음 → `IGraphSeries` 기반으로 일원화
* `SeriesRendererManager`는 각 시리즈 타입에 맞는 `SeriesRendererBase<T>` 인스턴스를 동적으로 생성
* `RequestRender()` 호출 방식 제거 → 렌더 스레드가 지속적으로 자동 렌더링 수행
* `ViewProjectionBuffer`는 프레임당 1회만 업로드하는 공유 리소스로 전환됨
* `SetTransform()` 메서드는 더 이상 사용되지 않음

### 🎨 렌더링 구조 개선

* `RealTimeSeriesRenderer`는 내부적으로 셰이더와 GPU 버퍼를 초기화
* `ID3D11Device` 및 `ID3D11DeviceContext`는 `SeriesRendererManager` → `SeriesRendererBase`로 직접 전달됨
* `DrawGroupBatcher`는 동일한 `DrawKey` 기준으로 시리즈 렌더러를 그룹화하여 `Upload`, `Draw`를 배치 수행
* 렌더러 내부에서 셰이더 바인딩 책임을 갖도록 수정함 → D3D11Renderer는 더 이상 셰이더에 관여하지 않음

### 🔍 테스트 및 검증

* `RealTimeSeries<VertexPosition>`을 이용하여 삼각형 정점 시각화 테스트 수행
* 삼각형이 우측 상단에 보이는 문제를 뷰포트 스케일 조정으로 해결함
* 버텍스 카운트가 정상적으로 전파되는 것까지 확인 완료

### 🧹 기타

* 모든 GraphItem 관련 코드 및 인터페이스 제거
* `SetTransform()`과 관련된 업데이트 호출 제거

---

이 변경을 기준으로 고성능 실시간 렌더링 파이프라인의 기본 골격이 완성됨. 이후 작업은 Axis, Cursor, Marker 등 시리즈화, ShaderManager 도입으로 이어질 예정.
