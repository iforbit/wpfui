## ✅ 오늘의 주요 작업 내역

### 🧪 1. SharedMemory 기반 실시간 수신 루프 개선

* `StartPolling()` 루프의 타이밍 정밀도 향상을 위해 `Stopwatch`를 도입하여 정확한 10ms 주기를 유지하도록 조정
* 중복 데이터 필터링 로직 강화 (NaN 방지 및 값 변화량 필터 적용)
* 수신 루프의 정확한 주기와 signal hz 간의 불일치를 감지하고 개선 방향 검토

### 🔍 2. SignalEngine x축 기준 변경

* x값을 0.001 간격의 누적 시간에서 `Stopwatch.ElapsedMilliseconds`로 변경하여 **실제 시간(ms)** 기반 x축 구성
* x값 단위 변경에 따라 `XScale` 조정 필요성 도출됨 (초당 100\~200 포인트 기준으로 적정 뷰포트 구성 검토)

### 🎯 3. 시리즈 구조 재검토 및 다중 시리즈 타입 설계

#### 🔹 기존 구조의 한계

* 렌더링 스레드(RenderThreadService)가 지연될 경우 실시간 그래프에서도 뷰의 표현이 밀리거나 늘어지는 현상 발생
* `RealTimeSeries` 구조에서는 입력 간격이 뷰 너비에 직접 영향을 주므로 주기 변화나 지연이 그대로 뷰에 반영됨
* 해결을 위해 `ViewTimeSeries`가 도입되며, 시간기준으로 고정된 뷰포트를 유지하고 점만 적절히 배치

#### 🔸 해결 방안: **시리즈 타입을 목적에 따라 분리**

| 타입                   | 시간 기준          | 목적         | 적합한 상황                 |
| -------------------- | -------------- | ---------- | ---------------------- |
| `RealTimeSeries<T>`  | 외부 입력 x값       | 정확한 시계 동기화 | SCADA, 로그 재현, 복수 장비 비교 |
| `ViewTimeSeries<T>`  | 수신 시간          | 단순 실시간 시각화 | UI 시계 기반 표현            |
| `FixedRateSeries<T>` | 내부 타이머 (고정 간격) | 균일 간격 시각화  | 시뮬레이션, 분석, 샘플링 보정      |
| `BufferedSeries<T>`  | 비동기 Flush      | 대용량 효율 처리  | 서버 수신, 버퍼 최적화          |

#### ✅ 시리즈 분리 및 설계 확정

* 기존 `RealTimeSeries<T>`만 존재하던 구조에서 다음과 같이 시리즈의 목적과 표현 방식에 따라 분리 작업 시작:

```
IGraphSeries (공통 인터페이스)
├─ IGraphDataSeries<T>         ← 실제 데이터를 가지는 시리즈만
│   ├─ RealTimeSeries<T>       ← x값 기반, 데이터가 입력되는 시간 간격 그대로 그려짐
│   └─ ViewTimeSeries<T>       ← 시간 축이 고정되어 있고, 정해진 시간 슬롯에 값을 할당하여 그리는 방식
└─ IGraphAuxiliarySeries       ← 축, 커서 등 보조 렌더링 요소들
    ├─ AxisLineSeries
    └─ CursorLineSeries
```

* 각 시리즈는 내부적으로 `SeriesRendererBase<T>`와 매핑되며, `SeriesRendererManager`가 자동 분기 생성
* `DrawGroupBatcher`는 여전히 `IGraphSeries`를 기준으로 DrawKey 병합 처리
* `IGraphDataSeries<T>`는 다음 인터페이스 명세로 분리:

```csharp
public interface IGraphDataSeries<T> : IGraphSeries where T : unmanaged
{
    void Append(ReadOnlySpan<T> data);
    ReadOnlySpan<T> GetSpan(); // 또는 범위 기반 호출 지원
}
```

* `RealTimeSeries`는 입력된 x값이 시간 의미를 가지며, 연속 데이터 간격이 불균일한 경우 뷰 크기와 스케일링이 영향받음
* `ViewTimeSeries`는 내부 기준 시간(예: Stopwatch)에 따라 뷰포트를 고정하여 일정 간격으로 데이터를 배치

  * 예: 1000ms → 100픽셀 가정 시, 1ms = 0.1px → 데이터 도착 시간과 무관하게 균등 배치 가능
#### 🔄 GraphControl, TransformManager 영향도 검토

* 현재 스케일 자동조정 로직은 데이터 간격 기반(`AutoScaleHelper.EstimateXScale`)으로 구현되어 있음
* `RealTimeSeries` 전용 동작이며, `ViewTimeSeries` 적용 시 TransformManager는 일정 스케일로 고정된 뷰포트 유지 필요

#### 🧱 향후 구현 계획

* `SeriesType` 혹은 팩토리 기반 등록 구조 설계 예정
* `GraphControl` 또는 `SeriesRendererManager`에서 시리즈 타입에 따라 적절한 렌더러 자동 배정
* `AutoScroll`, `XRangeLock`, `TimeAlignMode`와 같은 고급 설정 속성 확장 고려
* [ ] `ViewTimeSeries<T>` 구현 및 렌더러 확장
* [ ] `IGraphAuxiliarySeries` 인터페이스 정의 및 기존 커서/축 통합
* [ ] `AutoAdjust` 모드에서 X축 범위가 고정일 경우 처리 방식 정립
* [ ] 실시간 Signal 장비에서 시간 오차가 존재할 경우 두 시리즈의 차이 비교 및 성능 측정


### 📌 기타 메모

* `AutoScaleHelper.EstimateXScale()`는 현재 미사용 상태이나, `RealTimeSeries`에만 적용 가능함
* `ViewTimeSeries`는 TransformManager 또는 GraphControl에서의 x축 관리를 요구하므로 별도 `TimeWindowSec` 속성 고려 필요
