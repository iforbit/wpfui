#### 🚀 Writer/Reader SharedMemory 통신 구조 구축

**목표:** IBA TFR 스타일 실시간 신호 생성/수신 시스템 개발

##### 1. Demo.Writer 프로젝트 (신호 생성기)

**구조:**
```csharp
SharedMemoryWriter
├─ Header: 12 bytes
│  ├─ channelCount (int, 4 bytes)
│  ├─ pointsPerChannel (int, 4 bytes)
│  └─ updateCounter (int, 4 bytes)  ← 중복 수신 방지용
├─ Data: maxChannels × maxPointsPerChannel × 12 bytes
│  └─ (double X, float Y) per point
└─ 총 버퍼 크기: 12 + (10 × 50,000 × 12) = 6,000,012 bytes
```

**SignalGenerator 구현:**
- 6가지 파형 타입: `Sine, Cosine, Square, Triangle, Sawtooth, Noise`
- 파라미터: `Frequency, Amplitude, Phase, Offset`
- 샘플 인덱스 기반 X축 생성 (0, 1, 2, 3, ...)
- `GeneratePointsWithIndex(startIndex, sampleRate, pointCount)`:
  ```csharp
  // X = 샘플 인덱스
  double xValue = startIndex + i;

  // Y = 샘플 인덱스 / 샘플레이트로 시간 계산
  double time = (startIndex + i) / sampleRate;
  float yValue = Generate(time);
  ```

**Frequency 계산 방식:**
```csharp
// PointsPerUpdate 개수로 Frequency 횟수만큼 사이클 생성
sampleRate = config.Frequency * _pointsPerUpdate;

// 예: Frequency=1Hz, PointsPerUpdate=5000
//     → sampleRate=5000 → 5000개 포인트로 정확히 1 사이클
// 예: Frequency=2Hz, PointsPerUpdate=5000
//     → sampleRate=10000 → 5000개 포인트로 정확히 2 사이클
```

**WriteBoardViewModel:**
- `UpdateRate`: 60Hz (1초에 60번 업데이트)
- `ChannelCount`: 1~10 채널 동적 설정
- `PointsPerUpdate`: 10~50,000 포인트
- **실시간 UI 반영**: 모든 NumberBox에 `UpdateSourceTrigger=PropertyChanged` 적용
- **한글 툴팁**: 각 파라미터 상세 설명 추가

**UI 구성:**
```xaml
<ui:CardControl> Global Settings
  ├─ Update Rate (Hz): 초당 전송 횟수
  ├─ Channel Count: 채널 개수 (1~10)
  └─ Points Per Update: 한 번에 보낼 포인트 수

<ui:CardExpander> Channel Configuration (ScrollViewer)
  ├─ Signal Type: ComboBox (6가지 파형)
  ├─ Frequency (Hz): 초당 반복 횟수
  ├─ Amplitude: 파형 세로 크기
  ├─ Phase (radians): 좌우 이동 (0~2π)
  └─ Offset: Y축 이동
```

##### 4. 구현 과정에서 해결한 이슈

**Issue 1: Frequency 계산 오류**
- 증상: Frequency=1Hz로 설정해도 1 사이클이 안 나옴
- 원인: `sampleRate = _updateRate * _pointsPerUpdate` (시간 기반)
- 해결: `sampleRate = config.Frequency * _pointsPerUpdate` (포인트 기반)

**Issue 2: 채널 개수 변경 시 이전 채널 유지**
- 증상: 2채널 → 1채널 변경 시 CH2가 계속 보임
- 원인: 하드코딩된 `_channels = ["CH1", "CH2"]` 배열
- 해결: `UpdateChannels(channelCount)` 동적 채널 관리

**Issue 3: UI 입력값 미반영**
- 증상: NumberBox에 값 입력 후 포커스를 잃어야 반영됨
- 원인: 기본 바인딩 모드 `UpdateSourceTrigger=LostFocus`
- 해결: 모든 NumberBox에 `UpdateSourceTrigger=PropertyChanged` 추가

**Issue 4: Y축 떨림 현상**
- 증상: 그래프가 계속 상하로 흔들림
- 원인: `AutoAdjustTransform()`이 최근 500개 포인트만 보고 Y 범위 계산
- 해결: `SetYScaleLock(true)`로 Y축 고정

**Issue 5: SharedMemory 오프셋 불일치**
- 증상: 1줄만 그려짐 (다채널 안 됨)
- 원인: Reader `baseOffset = 8 + channelIndex * pointCount * 12` (가변)
- 해결: Writer와 동일하게 `baseOffset = 12 + channelIndex * maxPointsPerChannel * 12` (고정)

**Issue 6 (부분 해결): 그래프 직선 구간 발생**
- 증상: 일정 간격으로 직선이 그려지는 구간 발생
- 원인: Reader가 Writer보다 빠르게 읽어서 중복 데이터 필터링 시 전체 스킵
- 해결 시도:
  - 폴링 간격 10ms → 16ms로 조정
  - Header에 updateCounter 추가 (미완성)
- **향후 개선 필요**: updateCounter 기반 중복 감지 로직 완성

#### 📊 성능 특성

**데이터 처리량:**
- Writer: 60Hz × 1000 points = 60,000 points/sec per channel
- Reader: 16ms 폴링 → ~62.5 FPS
- SharedMemory: 6MB 고정 버퍼

**메모리 효율:**
- RingBuffer: GC-free circular buffer
- ArrayPool: 대용량 버퍼 병합 시 풀링
- Span<T>: Zero-copy 데이터 전달

**렌더링:**
- DrawGroupBatcher: DrawCall 최소화
- GetSpanMergedUnsafeCached(): 끊김 없는 wrap-around 렌더링
- GPU 상수 버퍼: 프레임당 1회 업로드
